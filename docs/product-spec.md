# TripWise Planner 产品说明

## 产品定位

TripWise Planner 是一个面向懒得做攻略的用户的自动旅行规划工具。用户只需要输入出发地、日期、目的地、预算和偏好，系统自动生成多城市路线、跨城交通、每日行程和预算拆分。

## MVP 用户流程

1. 用户输入出发地、出行日期、目的城市、预算和可选酒店/住宿名。
2. 用户选择旅行风格：均衡、省钱、省时间、少折腾。
3. 系统生成三套方案：均衡、省钱、省时间。
4. 用户查看城市顺序、跨城交通、每日安排和预算结构。
5. 用户调整目的地或偏好后重新生成。

## 当前 MVP 能力

- 内置杭州、广州、深圳、茂名、澳门、佛山、珠海样例数据。
- 配置高德 Key 后，支持地理编码、POI 搜索、驾车路线估算和动态路线图。
- 支持路线图点开查看、每日行程详情弹窗、POI 图片、高德链接和小红书搜索跳转。
- 支持高铁、航班、自驾、大巴交通偏好，并展示多候选方式。
- 支持 12306、携程机票/汽车票、高德路线的外部查询入口。
- 支持填写已定酒店/住宿名，并估算酒店到首末景点、景点之间的日内交通时间。
- 使用启发式路线评分，综合考虑交通时间、交通成本和舒适度。
- 自动把目的城市分配到每日行程，并尽量避免同城多日 POI 重复。
- 自动生成跨城交通建议、每日住宿区域和更具体的景点活动；大范围目的地会按地区/片区拆分。
- 支持新增自定义城市。

## 后续工程方向

- 地图：继续接入真实城市内公交/步行/驾车路线、景点排序、营业时间和闭馆日。
- 票务：接入火车/机票实时班次和价格，当前先做查询跳转，不直接售票。
- 酒店：按预算、城市区域和每日片区推荐酒店位置。
- 优化：将 `src/planner.js` 的启发式算法升级为约束优化模型。
- 账户：加入收藏、历史方案、多人协同和导出能力。

## 推荐技术演进

当前版本已经升级为前后端 MVP，前端通过 `/api/plan` 调用后端规划服务。下一阶段可以迁移为：

```text
前端：React / Vue
后端：Node.js / Python
数据库：PostgreSQL
地图：高德地图开放平台
缓存：Redis
AI：大模型负责理解偏好、生成解释和改写行程
优化：路线规划引擎负责城市顺序、时间和成本计算
```
